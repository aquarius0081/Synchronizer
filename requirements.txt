Тестовое задание

Создать java приложение, выполняющее две основные функции:
V 1) выгрузка содержимого таблицы БД в XML файл;
V 2) синхронизация содержимого таблицы БД по заданному XML файлу.

Общие требования
V •	Основные этапы операций должны логироваться в лог файл. На экран должна выводиться только краткая информация о результате прохождения операции.
V •	Название и местонахождение файла для логирования, а также параметры соединения с БД должны задаваться в файле настроек приложения. Формат файла – java properties.

Требования к таблице в БД
V В БД должна быть одна таблица со следующей структурой:
V ID int  – суррогатный ключ генерируемый при вставке записи в таблицу;
V DepCode String(длина 20 символов) – код отдела;
V DepJob String(длина 100 символов) – название должности в отделе;
V Description String (длина 255 символов) – комментарий.
V Первичный ключ – ID.
V Натуральный ключ состоит из двух полей – DepCode, DepJob. Соответственно в таблице не должно быть двух и более записей с одинаковым сочетанием DepCode и DepJob.

Требования к функции выгрузки содержимого таблицы в XML файл
V •	Пользователь должен иметь возможность указывать имя файла, в который будет осуществлена выгрузка.
V •	Формат получаемого файла должен быть – XML.
V •	В полученный файл должны выгружаться все поля таблицы, перечисленные выше, кроме поля ID.

Требования к функции синхронизации содержимого таблицы c заданным XML файлом 
V •	Пользователь должен иметь возможность указывать имя файла по содержимому которого будет осуществляться синхронизация данных в таблице.
V •	Структура загружаемого файла должна быть такой же, как и у выгружаемого в функции выше. Т.е. приложение должно корректно загрузить выгруженный ранее файл.
V •	Синхронизация должна проходить по натуральному ключу таблицы с минимальным числом обращений к БД. Т.е. удаляться должны только те записи, которые отсутствуют в XML файле. Добавляться соответственно только новые. Измененные должны обновляться.
V •	В случае если в XML файле есть две записи с одним натуральным ключом, то приложение должно выдать соответствующую ошибку.
V •	Весь процесс синхронизации должен осуществляться в одной транзакции, т.е. если в процессе синхронизации произошла ошибка, то данные в БД должны остаться нетронутые.

Требования к используемым технологиям
V •	Доступ к СУБД должен осуществляться посредством JDBC.
V •	Обновление данных в БД должно осуществляться с помощью SQL выражений, а не CONCUR_UPDATABLE режима.
V •	Логирование в файл должно осуществляться посредством библиотеки log4j
V •	В алгоритме синхронизации должен быть использован класс HashMap или HashSet. Для контроля ключей рекомендуем создать специальный класс, в котором реализовать методы хеширования и equals.
V •	Чтение XML файла должно осуществляться с помощью технологии XML DOM. Запись – на ваш выбор.
V •	Для работы с файлом настроек приложения необходимо использовать класс Properties.
V •	Задание выполняемой команды (функции), а также имени файла рекомендуем сделать через параметры командной строки. Пример: test.bat sync test.xml

Требования к оформлению
V 1.	Должен быть предоставлен собранный и готовый к запуску проект в виде zip архива. Архив должен обязательно содержать:
V •	файл readme.txt, в котором нужно кратко описать, что необходимо выполнить перед запуском проекта;
V •	бакап БД;
V •	файл настроек приложения;
V •	bat скрипт запуска приложения.
V 2.	Должны быть предоставлены все исходные коды в виде zip архива. Исходный код Java должен быть аккуратно оформлен и документирован в соответствии с технологией JavaDoc.

Неуказанные детали реализуйте по вашему выбору.
